## Live Sync

Two transports, same cursor:

- **Long poll:** `GET /api/projects/:pid/events?since=<cursor>` — returns up to N events or waits up to 30s for new ones. Good for proxies that strip SSE; the engine plugin's default.
- **SSE:** `GET /api/projects/:pid/events?stream=sse` — continuous stream. Browser's default ([ADR-027 Engine-plugin sync is snapshot + event-stream, server-authoritative](/decisions/027-engine-plugin-sync-protocol)).

```mermaid
sequenceDiagram
  participant C as Client (browser / plugin / CLI)
  participant B as Backend
  participant DB as entity_versions table

  Note over C: first run or stale mirror
  C->>B: GET /api/projects/:pid/snapshot
  B->>DB: stream all entity rows + latest lsn
  DB-->>B: rows
  B-->>C: NDJSON entities + cursor

  loop live sync
    C->>B: GET /events?since=cursor
    B->>DB: SELECT lsn > cursor
    alt changes pending
      DB-->>B: events
      B-->>C: entity.updated / deleted / asset.committed / schema.changed / proposal.* / agent.run.*
      C->>C: apply to cache, advance cursor
    else idle (30s)
      B-->>C: heartbeat + same cursor
    end
  end

  alt cursor older than retention window (410 Gone)
    B-->>C: { error: "cursor-too-old", snapshotHint: "/snapshot" }
    C->>B: GET /snapshot
    Note over C: rebuild mirror from cold snapshot
  end
```
