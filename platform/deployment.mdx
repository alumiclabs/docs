## Overview

Under SaaS-first ([ADR-001 SaaS-first, on-prem second](/decisions/001-saas-first)), the backend is the **primary data layer**, not a Phase 4 add-on. A single Fastify + Postgres + MinIO stack serves all three deployment modes — hosted SaaS, on-prem, single-user desktop — from one codebase. This chapter documents:

1. **The Entity API** — the central HTTP surface for all authored content.
2. **Authentication and identity** — session JWTs for browsers, PATs for plugins/CLI/CI.
3. **Sync and push** — the incremental event stream that keeps clients in sync.
4. **Multiplayer** (`@alumic/multiplayer`) — the Colyseus-based runtime networking layer, orthogonal to the authoring API.

## Deployment packaging

The backend lives in `packages/backend/` inside the same Bun workspace as the rest of Alumic ([ADR-001 SaaS-first, on-prem second](/decisions/001-saas-first)). It is not a separate repo.
